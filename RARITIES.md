# Rarity Tiers Documentation

This document describes all 12 rarity tiers in the Rarity and Relics mod, their properties, and drop rates.

## Overview

The mod features **12 distinct rarity tiers**, each with unique:
- **Color scheme** for item names
- **Description** reflecting item quality
- **Drop rates** for different sources (crafting, mob drops, loot)
- **Modifier count** scaling with rarity
- **Modifier strength** multiplier

## Rarity Progression System

### Modifier Scaling

Higher rarity items receive:
1. **More modifiers** (0-12 based on tier)
2. **Stronger modifiers** (0.3x to 5.0x multiplier)
3. **Better stat ranges** (scaled from base -20% to +30%)

### Drop Rate System

Three independent drop rates control where items appear:
- **Crafting**: When players craft items at crafting tables
- **Drop**: When mobs are killed and drop equipment
- **Loot**: When items are found in chests, structures, etc.

## Probability Analysis

### Crafting Distribution

When crafting items, you have:
- **60%** chance: Common
- **20%** chance: Uncommon
- **10%** chance: Rare
- **5%** chance: Epic
- **2%** chance: Legendary
- **1%** chance: Mythic
- **<1%** chance: Ancient, Celestial, Divine, Transcendent
- **~0.1%** chance: Trash (bad luck!)
- **0%** chance: Owner

### Combat Drops Distribution

When killing mobs, the distribution shifts slightly:
- Higher tiers slightly more common than crafting
- Ancient/Celestial/Divine/Transcendent more accessible
- Still heavily weighted toward Common/Uncommon

### Loot Chest Distribution

Chests use the same rates as mob drops, encouraging exploration.

## Configuration

All these values can be customized in:
```
config/rarity-and-relics/rarities.json
```

You can:
- Add new rarity tiers
- Adjust drop rates
- Change colors
- Modify descriptions
- Create custom asset references

### Adding a Custom Rarity

```json
{
  "id": "custom_tier",
  "name": "Custom Tier",
  "color": "#FF00FF",
  "description": "Your custom description.",
  "assets": {
    "icon": "rarity:icons/custom_tier",
    "frame": "rarity:frames/custom_tier"
  },
  "dropRates": {
    "crafting": 5.0,
    "drop": 5.0,
    "loot": 5.0
  }
}
```

## Technical Implementation

### Source Type Enum

The mod uses three source types internally:
```java
public enum SourceType {
    CRAFTING,  // Crafting tables
    DROP,      // Mob kills
    LOOT       // Chests/structures
}
```

### Weighted Selection

Rarities are selected using weighted random:
1. Sum all drop rates for the source type
2. Generate random value [0, total]
3. Iterate through rarities, accumulating weights
4. Return rarity when accumulated weight exceeds random value

### Modifier Generation

For each item:
1. Select rarity based on source type
2. Determine modifier count from rarity tier
3. Generate N random modifiers
4. Apply rarity multiplier to base range
5. Store in RarityData component

## Advanced Topics

### Balancing Considerations

1. **Drop Rates**: Total should sum to ~100% for best distribution
2. **Modifier Counts**: Higher tiers should have more modifiers
3. **Multipliers**: Scale exponentially, not linearly
4. **Source Differentiation**: Consider making rare tiers more accessible through specific sources

### Future Enhancements

Possible additions:
- **Set bonuses**: Matching rarity equipment provides bonus
- **Reroll mechanics**: Items to change rarity
- **Rarity upgrading**: Combine items to increase tier
- **Special effects**: Visual particles for high rarities
- **Sound cues**: Audio feedback on rare drops
